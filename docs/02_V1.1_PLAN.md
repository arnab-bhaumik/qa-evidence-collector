# QA Evidence Collector — V1.1 Plan: Image Annotation

## Overview

V1.1 adds image annotation capabilities to QA Evidence Collector. After capturing a screenshot, the tester can annotate it before saving — drawing arrows, adding text labels, highlighting areas, and blurring sensitive data. The annotated image replaces the original in both the step view and the generated DOCX report.

---

## Motivation

During manual testing, testers often need to point out specific areas of a screenshot — a button that was clicked, an error message, a field that contains wrong data. Without annotation, reviewers must guess what each screenshot is showing. Annotation makes evidence clearer, more professional, and easier to review.

---

## New User Workflow (V1.1)

1. User captures a screenshot (button or hotkey)
2. **Annotation Editor opens** — user can draw on the image
3. User applies arrows, text, highlights, or blurs as needed
4. User clicks **Done** — annotation is saved
5. Note dialog opens as before
6. Annotated image appears in View Steps and in the DOCX report

---

## Features

| Feature | Description |
|---------|-------------|
| Arrow tool | Click and drag to draw a directional arrow pointing at a UI element |
| Text tool | Click anywhere on the image to place a text label |
| Highlight tool | Drag to draw a semi-transparent coloured rectangle around an area |
| Blur tool | Drag to pixelate/blur a region containing sensitive data |
| Undo | Step-by-step undo of annotations |
| Clear | Remove all annotations and start fresh |
| Done / Skip | Save annotated image or skip annotation entirely |

## S8 — Colour Picker Detail

Users can select a colour for each annotation tool before drawing. The selected colour applies to all new annotations drawn with that tool until changed.

### Colour Options Per Tool

| Tool | Applies To |
|------|-----------|
| Arrow | Shaft colour + arrowhead fill |
| Text | Text colour + background contrast auto-adjusts |
| Highlight | Fill + border colour (replaces the double-click preset cycle) |
| Blur | No colour — blur is always greyscale pixelation |

### UI Design

- A **colour swatch row** appears in the tool panel below the active tool button
- Shows **6 preset colour swatches** (red, orange, yellow, green, blue, purple)
- A **custom colour button** (`+`) opens the system colour picker (`QColorDialog`)
- Selected colour is shown with a white ring highlight
- Colour persists per tool — switching tools remembers each tool's last colour

### Preset Colours

| Swatch | Hex |
|--------|-----|
| Red | `#FF3B30` |
| Orange | `#FF9500` |
| Yellow | `#FFD600` |
| Green | `#34C759` |
| Blue | `#007AFF` |
| Purple | `#AF52DE` |

## Technical Approach

| Component | Technology |
|-----------|-----------|
| Annotation canvas | PySide6 `QGraphicsScene` + `QGraphicsView` |
| Arrow / highlight drawing | `QPainter`, `QGraphicsItem` subclasses |
| Text labels | `QGraphicsTextItem` |
| Blur effect | Pillow (`ImageFilter.GaussianBlur`) applied to cropped region |
| Saving annotated image | Pillow — composite annotations onto original PNG |
